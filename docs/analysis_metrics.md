# 分析指標定義書

> 新世界 S4 チッケム動画分析ダッシュボード（STAT PICK）向け  
> 最終更新: 2026-03-05

---

## 概要

本ドキュメントは、YouTube Data API v3 および Chrome拡張機能（YT Comment Exporter）で収集したデータをもとに算出する分析指標の定義・計算式・取得元を定めます。

---

## 1. 基本動画指標（GAS 毎日自動取得）

| 指標名（コード） | 計算式 / 内容 | 意味 |
|---|---|---|
| `viewCount` | YouTube API 取得値 | 再生回数（取得時点の累計） |
| `likeCount` | YouTube API 取得値 | いいね数（取得時点の累計） |
| `commentCount` | YouTube API 取得値 | コメント数（取得時点の累計） |
| `engagementRate` | `(likeCount + commentCount) ÷ viewCount` | 総合エンゲージメント率 |
| `likeRate` | `likeCount ÷ viewCount` | いいね率 |
| `commentRate` | `commentCount ÷ viewCount` | コメント率 |

### 日次スナップショット（`daily_log` に蓄積）

| 指標名（コード） | 内容 |
|---|---|
| `views_cum` | 再生回数（累計） |
| `likes_cum` | いいね数（累計） |
| `comments_cum` | コメント数（累計） |
| `views_delta` | 再生回数（前日比増加量） |
| `likes_delta` | いいね数（前日比増加量） |
| `comments_delta` | コメント数（前日比増加量） |

> **日付処理の注意：** `daily_log.date` は前日付け（実績日）で記録する。実際の取得日時は `recorded_at` 列に別途保存する。例えば3月5日02:30に取得したデータは `date = 2026-03-04`、`recorded_at = 2026-03-05 02:30:00 JST` となる。

## 2. コメント分析指標（Chrome拡張 週1回手動取得）

### 2-1. コメント参加・熱量指標

| 指標名（コード） | 計算式 | 意味 |
|---|---|---|
| `commentUU` | ユニークユーザー数 | コメントした人数（重複除外） |
| `commentParticipationRate` | `commentUU ÷ viewCount` | 視聴者のうちコメントする割合 |
| `avgCommentsPerUser` | `commentCount ÷ commentUU` | 1人あたり平均コメント数（熱量指標） |
| `maxCommentsBySingleUser` | 最大コメント数ユーザーの投稿数 | 最もコメントした1人の投稿数 |
| `commentDispersionRate` | `commentUU ÷ commentCount` | コメントの分散度。1に近いほど多くの人が1回ずつコメント |
| `commentGini` | ジニ係数 | コメント集中度（0=完全均等、1=1人独占）。全体の不平等度を統計的に示す |
| `commitmentRate` | `commentUU ÷ likeCount` | 好意を持った人（いいね）のうち実際にコメントした割合 |

> **`commentDispersionRate` vs `commentGini` の使い分け：**  
> `commentGini` は全体の偏りを示すメイン指標。`commentDispersionRate` は直感的な補助指標として併用する。両者は似た概念だが、ジニ係数は「全体の不平等度」、分散率は「トップ1人の突出度」で意味が微妙に異なる。

### 2-2. 言語・地域指標

| 指標名（コード） | 内容 |
|---|---|
| `lang_ja_rate` | 日本語コメントの割合 |
| `lang_en_rate` | 英語コメントの割合 |
| `lang_ko_rate` | 韓国語コメントの割合 |
| `lang_zh_rate` | 中国語コメントの割合 |
| `lang_other_rate` | その他言語コメントの割合 |
| `domestic_rate` | 国内（日本語）率 |
| `overseas_rate` | 海外率（日本語以外） |

> **注意：** 海外率はバイアスリスクがあるため、総合スコアには含めず参考情報として別表示とする。

## 3. トーン分類（Chrome拡張 週1回手動取得）

### 3-1. なぜ「トーン分類」なのか

ポジティブ／ネガティブの二値判定は、K-POPオーディション文脈では誤判定が多発する。「やばい」「死ぬ」「沼った」「殺される」はネガティブな単語だが、ファンダムでは強い称賛を意味する。英語の「I'm dead」「He ate」「Slay」も同様。

トーン分類は「感情の良し悪し」ではなく**「コメントの目的・種類」**で分類する。これにより炎上リスクを回避しながら有益なインサイトを得られる。

### 3-2. 6カテゴリ定義

| カテゴリ | コード | 内容 | ワード例 |
|---|---|---|---|
| 🎤 応援・推し表明 | `CHEER` | 投票呼びかけ、ファン宣言、デビュー希望、1Pick宣言 | 投票して、デビューして、私の1Pick、絶対、頑張れ |
| ✨ パフォーマンス評価 | `PRAISE` | ダンス・歌・ビジュアルへの具体的な言及 | ダンス上手い、歌声、ビジュアル、かっこいい |
| 💬 感想・印象 | `REACT` | 「感動した」「好きになった」など全体的な感想 | 泣いた、やばい（称賛）、好き、最高、神 |
| ❓ 質問・情報収集 | `QUESTION` | 年齢・事務所・SNSなど情報を求めるコメント | 何歳、事務所どこ、インスタある？ |
| 🌐 海外ファン | `OVERSEAS` | 日本語以外の言語のコメント（英/韓/中/その他） | （言語判定で自動分類） |
| ☁️ その他・未分類 | `OTHER` | 上記に該当しないもの（残余カテゴリ） | — |

> **K-POPファンダム特有の注意点：** 「やばい」「死ぬ」「沼った」「殺される」「I'm dead」「He ate」「Slay」は一般辞書ではネガティブだが、ファンダムでは強い称賛を意味するため専用処理が必要。

### 3-3. 分類優先順位（重複解消ルール）

複数カテゴリに該当する場合は以下の優先順位で分類する：

`CHEER` > `PRAISE` > `REACT` > `QUESTION` > `OVERSEAS` > `OTHER`

### 3-4. トーン指標

| 指標名（コード） | 計算式 |
|---|---|
| `tone_cheer_rate` | CHEERコメント数 ÷ 総コメント数 |
| `tone_praise_rate` | PRAISEコメント数 ÷ 総コメント数 |
| `tone_react_rate` | REACTコメント数 ÷ 総コメント数 |
| `tone_question_rate` | QUESTIONコメント数 ÷ 総コメント数 |
| `tone_overseas_rate` | OVERSEASコメント数 ÷ 総コメント数 |
| `tone_other_rate` | OTHERコメント数 ÷ 総コメント数 |

> **炎上検知：** `OTHER`の急増 + 特定ネガティブワードの検出でアラートを出す設計とする。ネガティブ率の急上昇（前週比2倍以上など）は要注意シグナルとして扱う。

## 4. テキスト分析指標（Pythonスクリプトによるスポット分析）

コンテンツ制作時の深堀り分析に使用。スプレッドシートには保存せず、ローカルで実行する。

| 指標名（コード） | 内容 | 手法 |
|---|---|---|
| `keyword_frequency` | 頻出キーワード | 形態素解析（janome / MeCab） |
| `keyword_cooccurrence` | 共起語 | 共起ネットワーク分析 |
| `bigram_frequency` | 2語頻度 | n-gram分析 |
| `sentiment_score` | 簡易感情スコア | ワードリストベース |

**実行フロー：**
1. Chrome拡張でCSVを取得（全動画 or 練習生別）
2. Pythonスクリプトで一括分析（`analysis/` ディレクトリ内のスクリプト）
3. 練習生ごとの特徴キーワード・共起語・感情スコアを出力
4. コンテンツ制作に活用

## 5. 総合スコア（仮確定）

**コメント欄での注目度スコア**として定義する。実際の投票数や人気の絶対値ではないことをダッシュボード上に明記する。

| 指標 | 重み | 根拠 |
|---|---|---|
| `likeRate`（いいね率） | **25%** | 1人1回・操作困難・行動の証拠 |
| `commentUU`（ユニーク参加者数） | **25%** | 実際にコメントした人数・水増し不可 |
| `avgCommentsPerUser`（平均コメント数） | **20%** | ファンダムの熱量・深さ |
| `tone_cheer_rate`（応援率） | **15%** | 投票意欲の強さ・番組目的との直結 |
| `tone_praise_rate`（パフォーマンス評価率） | **10%** | 実力への注目度 |
| `commentCount`（コメント数絶対量） | **5%** | 関心の絶対量（重みは最低限） |
| `overseas_rate`（海外率） | **除外** | バイアスリスクあり → 参考情報として別表示 |

> **注意：** 応援率・パフォーマンス評価率はトーン分類のワードリスト精度に依存する。ワードリストが充実するほどスコアの信頼性が上がるため、運用しながら随時ワードを追加・調整できる仕組みにする。

## 6. ファンダムタイプ診断（3軸モデル）

### 6-1. 3軸の定義

| 軸 | 記号 | 使用指標 | 判定方法 |
|---|---|---|---|
| 第1軸：リーチ（Reach） | R / D | `commentCount`（全練習生中の相対順位） | 全練習生の中央値で二分 |
| 第2軸：アクティブ度（Activity） | A / W | `avgCommentsPerUser`（1人あたりの熱量） | 全練習生の中央値で二分 |
| 第3軸：分散度（Spread） | S / B | `commentDispersionRate`（%） | 全練習生の中央値で二分 |

### 6-2. 8タイプ分類

| タイプコード | 日本語名 | 英語名 | 特徴 |
|---|---|---|---|
| RAS | 拡散型 | Viral Force | 広く・活発に・分散している |
| RAB | 熱狂型 | Fandom Core | 広く・活発に・集中している |
| RWS | 静観型 | Silent Majority | 広く・静かに・分散している |
| RWB | 固定型 | Loyal Base | 広く・静かに・集中している |
| DAS | 原石型 | Hidden Gem | まだ知られていない |
| DAB | 深推し型 | Inner Circle | 深く推すほど好きが増していく |
| DWS | 潜在型 | Sleeper | 静かに広がっている |
| DWB | コア型 | Deep Niche | 少数精鋭の深いファン |

## 9. 制約・注意事項

- YouTube Data API v3 の公開データのみを使用するため、非公開情報（視聴者の属性、正確なインプレッション数など）は取得不可。
- コメント数は YouTube 側の仕様により、実際の投稿数と若干の誤差が生じる場合がある。
- コメントが無効化されている動画は、コメント分析の対象外となる。
- APIクォータ制限（1日10,000ユニット）に注意。大量の動画を処理する場合は分割実行を検討する。
- 同一動画が複数プレイリストに含まれる場合、`playlist_id` は優先度の高いプレイリストのIDが設定される。
- コメントローデータはスプレッドシートに保存しない（PCスペックへの配慮・スプシ軽量化のため）。ローカルCSVとして保管する。
